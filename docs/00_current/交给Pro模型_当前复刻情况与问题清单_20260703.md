## 7. 当前思考：冗余 vs 披露具体性

我们担心当前 Redundancy 与“披露具体性”概念混在一起。

概念区分：

```text
披露具体性 = 文本里有多少可核验的硬信息
信息冗余 = 文本中有多少可以被压缩、删掉而不损失核心信息的水分
```

可能的四象限：

| | 低冗余 | 高冗余 |
|---|---|---|
| 高具体性 | 理想披露：具体且紧凑 | 细节多但啰嗦、重复 |
| 低具体性 | 短但空泛 | 又空泛又啰嗦 |

如果后续把 Redundancy 当作新的 Y，可能需要同时构造 `Specificity`，再使用：

```text
净冗余 = Redundancy 在控制 Specificity 后的残差
```

但 strict reproduction 阶段，仍应先尝试复刻原文 Redundancy。
