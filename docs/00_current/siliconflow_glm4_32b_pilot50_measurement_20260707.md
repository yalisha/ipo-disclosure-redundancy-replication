# GLM4 dewrap_join 50 家 LLM measurement gate

日期：2026-07-07

## 结论

`NO_PASS_YET`

这轮只是在新的 `dewrap_join + GLM-4 tokenizer + 4000 token boundary split` chunk base 上试跑 50 家，检验量级是否能回到原文附近；它不是正式 gate。

核心读法：

- 50 家 821 个 chunk 已全部跑完 `cot_v3b_len132_tight`。
- 若沿用原项目此前的摘要长度 proxy，`Summary_len` mean=124.325，低于原文 132.678；`Redundancy_chunk` mean=35.731，高于原文 32.176。
- 若摘要也改用 GLM-4 tokenizer 计数，`Summary_len` mean=143.071，高于原文 132.678；但 `Redundancy_chunk` mean=31.362，反而更贴近原文 32.176。
- Panel B 在 50 家样本里方向为负：proxy 口径 rho=-0.093, p=0.0083；低评分组冗余中位数 32.765 高于高评分组 31.091，硬方向成立。
- 但 Spearman 强度远低于原定 gate 目标 `rho <= -0.45`，且 cluster 回归不显著；因此不能据此替换当前主 X。
- `Relevant_score` 有 13 个 chunk 缺失，原因是 GLM 返回的 `n0`-`n5` 全为 0；这些 chunk 已保留摘要，但不纳入评分相关性。
- Panel C / D 在 50 家样本里仍需谨慎解读；这里只看方向和是否出现明显异常。

## 判断

1. `dewrap_join` 新 chunk base 的分块数量和正文长度已经非常接近原文；这一步值得继续。
2. 50 家结果推翻了 5 家 pilot 的乐观量级：proxy 口径下 GLM 摘要偏短、chunk 冗余偏高；GLM-token 摘要口径下 chunk 冗余反而更贴近原文，但企业层 `Redundancy` 偏低。
3. Panel B 的方向和低/高评分中位数硬条件成立，但相关强度太弱，不能视为 measurement gate 通过。
4. Panel D 对创新词典为显著负向，方向正常；Panel C 为弱正，仍未提供额外支持。
5. 下一步不建议直接全样本 GLM。更合理的用法是：保留当前主 X，GLM 只作为模型差异稳健性或摘要长度单位诊断；若要继续 GLM，应先修评分输出格式，并明确 `Summary_len` 采用 proxy 还是 GLM-token 口径。
