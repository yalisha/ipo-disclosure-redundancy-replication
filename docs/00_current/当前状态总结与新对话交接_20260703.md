## 当前判断

### 已基本排除

- 不应继续优先调 prompt 或尾部修复。full scoregate 已把可合并 tail 修完，X 分布也明显降噪，但表 2 主结论仍不动。
- 不应简单认为是 2023 年样本稀释。2019-2022 窗口 N 贴原文，但系数仍没有复刻。

### 仍待核查

更可能的问题在：

1. 原文精确样本口径：是否只保留特定状态企业，是否剔除未盈利、特殊上市路径、特定行业、上市失败/撤回/无交易企业等。
2. `FInvention` 口径：授权 vs 申请、境内外、上市后第几期、是否按发明专利单项还是总专利、是否 winsorize 或缩放。
3. `BHAR` 口径：是否包含首日、窗口长度、市场基准、复权收益、停牌/缺失处理、是否用价值加权。
4. `FSales_Growth` 口径：营业收入字段、目标年度、缩尾、负收入/极端值处理。
5. 控制变量与固定效应：行业 FE 层级、上市年份 FE、规模/年龄/ROA/Lev/发行费用/承销商声誉是否与原文一致。
6. 定价/Panel D 还缺原文 `RD_Staff`，目前用 `RD_Asset` 只是 smoke test。

## 建议下一步

优先做一个“原文表 2 变量口径审计”小任务，不再动 LLM：

1. 从原文方法和变量定义表中逐项抄出 Table 2 的样本筛选、Y、controls、FE、缩尾口径。
2. 对当前 master 生成一张变量对账表：当前字段、来源、原文定义、是否一致、差异风险。
3. 对三个 Y 分别跑口径敏感性：
   - FInvention：授权/申请、境内/境外、`ln(1+x)`、不同目标年。
   - BHAR：含首日/剔首日、VW/EW market、窗口长度。
   - FSales_Growth：收入字段和缩尾口径。
4. 若口径审计后仍复不出，再判断这篇复刻是否应止损，或把 Redundancy 改作本项目自构造 Y / 机制变量。
